## Parsed checklist

1. **Rust: semantic variable names** — refactor `core/src/*.rs` so locals are
   descriptive (e.g. `compressed_data`, `output_buffer`, `file_size_human`)
   instead of generic `data`/`result`/`buf`.
2. **CI: add a `core` job** (name is exactly `core`, NOT `launcher-core`) that
   runs only when a file under `core/` is changed (`paths:` filter).
3. **Release attachments** — make sure the release uploads:
   - `*.snap`
   - `*.nix`
   - `*.flatpak`
   - every dotfile (`*`)
   alongside the existing binaries.
4. **Tag & push `v111.0.2`** (force the tag onto the new commit).
5. **Clean up GitHub Actions** — delete every failed run via `gh`.
6. **Directory layout** — follow the C++ standard `src/ lib/ res/` etc.;
   update all paths accordingly.
7. **Data vs code separation** — keep data out of source; data lives in
   `.yaml` / `.ini` files (loaded at runtime).
